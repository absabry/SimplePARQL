SimplePARQL: a new approach using keywords over SPARQL to query the web of data

Sonia Djebali & Thomas Raimbault, in proceedings of the 11th International Conference on Semantic Systems (SEMANTiCS), Vienna, Austria, 2015, 188-191, ACM.

Abstact: The SimplePARQL is a new and intuitive approach to query the Web of Data through existing SPARQL endpoints by using keywords in addition to SPARQL elements. Thus, the user is able to write more expressive pseudo-SPARQL queries where knowing the ontology (classes and properties) and resourcesâ€™ identifiers from an RDF base are not required. Concretely, a SimplePARQL query is transformed into N valid SPARQL queries that extend the initial query in order to reach the IRIs or literals in the RDF bases corresponding to keywords.

SimplePARQL query example

Looking for the movies where Brad Pitt starred?

With SimplePARQL With only SPARQL
For all bases:

SELECT *
WHERE
{
?Movies star "brad pitt"
}


Note:
For each term star and (brad pitt), our platform will attempt to match a specific resource which in one way or another is linked to corresponding term. In other words, it is not necessary to know identifiers (like <http://dbpedia.org/resource/Brad_Pitt>) of each handled resource.
On DBpedia:
SELECT *
WHERE
{
?Movies <http://dbpedia.org/property/starring> <http://dbpedia.org/resource/Brad_Pitt>
}

On Wikidata:
SELECT *
WHERE
{
?Movies <http://www.wikidata.org/entity/P161> <http://www.wikidata.org/entity/Q373912>
}
